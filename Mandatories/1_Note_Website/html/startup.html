Oprettelse af et projekt

Her bliver der beskrevet præcis hvad man skal bruge for at starte et projekt op, og derefter komme igang med at bygge sin webapplikation.

Node JS tilbyder flere forskellige måder at gøre tingene på, men der vil her blive taget udgangspunkt i hovedsageligt én enkelt måde, hvorefter man selv kan undersøge og finde alternative måder at komme gennem oprettelsen af projektet.

Der vil i enkelte tilfælde blive beskrevet forskellige måder at opsætte forskellige ting på (fx oprettelse af package.json).

Udviklingsmiljø

I denne gennemgang af Node JS bruges IntelliJ som integreret udviklingsmiljø (IDE). Det er dette miljø som vi hovedsageligt har arbejdet i gennem skoletiden, og derfor også oplagt at fortsætte med at benytte.

Her er links til IntelliJ og et par andre mulige IDE'er som kan bruges med Node.js:

  • Visual Studio
  • Eclipse
  • Andre IDE'er

    • Det skal dog lige siges at der i denne gennemgang ikke vil være beskrevet hvordan de andre IDE'er skal konfigureres for at komme igang med projektet.

    Dependencies

    Node Package Manager (NPM) bruges til at tilføje forskellige dependencies til éns projekt.

    Node JS er nemlig et lightweight framework, hvor man selv skal installere de nødvendige dependencies.
    Dette betyder også at projektets størrelse bliver så kompakt som muligt, da der ikke er alle mulige uønskede features som man muligvis ikke vil gøre brug af - og som derfor bare vil ligge og fylde unødvendigt.

    Disse dependencies bliver allesammen tilføjet til en fil der kaldes 'package.json'.
    Det er også denne fil som skal oprettes som det første, når man opretter et NodeJS projekt.

    Den kan laves på to forskellige måder:

  • Skriv 'npm init' i terminalen, og følg proceduren.
  • Opret den manuelt i roden af mappen på projektet.

    • Gøres det med 'npm init', bliver man guidet igennem de forskellige indstillinger - gøres det manuelt, skal man huske at gå ind i filen og tilføje { }, ellers kan den ikke genkendes som en json fil.

    Herefter kan man bruge npm kommandoer til at tilføje de dependencies som man igennem projektet skal bruge.

    Det vil også blive gennemgået i fremgangsmåden.

    OBS

    Windows brugere anbefales at ændre 'Shell path' til Powershell, da der i denne gennemgang vil blive brugt terminalen til at tilføje dependencies til projektet.
    Powershell kan hentes fra windows store.

    Ændring af shell path:

    Settings ➞ Tools ➞ Terminal ➞ Shell Path ➞ Skriv "powershell"

    Fremgangsmåde

    1. Åben IntelliJ og vælg 'Nyt projekt'
    2. Vælg 'Javascript' i venstre fane, og vælg derefter Javascript igen i den højre fane.
      "Empty project for developing apps using JavaScript, HTML, CSS, and Node.js."
    3. Opret en JS-fil i roden af mappen, der skal kaldes 'app.js'
      Dette vil være den fil som bruges til at lave get/post/put/delete mappings, og meget mere.
    4. Åben terminalen og sørg for at du er på stien til projektets rod.
      Mit projekt hedder '1_Note_Website', og derfor skal jeg være i mappen '..\1_Note_Website'



    5. Skriv 'npm init' i terminalen.
    6. Følg guiden (Hvis man trykker enter uden at indtaste noget, vil der blive brugt en standard.)
      ★ Vælg et passende package navn.
      ★ Version nummeret skal gerne matche konventionen (Starter med 0.0.1 ➞ 1.0.0 er når programmet er publiseret).
      ★ Indtast en passende beskrivelse til projektet.
      ★ Entry point skal sættes til den oprettede fil som blev kaldt 'app.js'.
      ★ Git repository er link til projektet på github.
      ★ Keywords er hvilke søgeord som der få projektet til at blive vist i søgningen.
      ★ License skal sættes til 'MIT'.

      Derefter vil package.json blive oprettet, og man kan åbne den for at se de indstillinger man har valgt.
